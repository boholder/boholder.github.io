#总结

#vrchat.com

官网域名 https://vrchat.com/home，同时下面有子域名 https://api.vrchat.com（虽然叫 api 但似乎只起到重定向到其他域名的作用）。

同时它曾经（2025 及以前？）承接 API，如 Unity 里的 VRChat SDK 所用到的 https://vrchat.com/api/1/file/image/code/{id}。不清楚目前 API 是否全面转为 api.vrchat.cloud，搜到一篇2026 年的文章还在用 https://vrchat.com/api/1/file/{fileId} 而不是 https://api.vrchat.cloud/api/1/file/{fileId}，尝试调用该接口也不会重定向。

本地开 dnscrypt-proxy DoH 代理（部分域名如 vrchat.com 会被 GFW 污染），dig 结果（其实是 HTTP 代理到日本的代理服务器再 DoH 到 Cloudflare 的 DNS 服务器，解析结果是 Cloudflare 提供的，下同、不再赘述）：

#pipeline.vrchat.cloud

VRChat’s Websocket API, also known as “the pipeline”, is used receiving updates regarding the API, such as receiving an invite request. The connection is receive-only, meaning that you can only listen for messages. Sending messages is undefined behavior. – Community Wiki 3

从游戏客户端日志里也能看到，客户端会尝试保持 Websocket 一直连接。根据社区 API 文档所提及的 API 功能，链接不稳定会导致游戏客户端无法显示最新状态，如好友在线情况不准、收不到各种请求通知。Websocket 是 TCP 长连接，如果这部分出、入双向流量选择走代理，则代理自身的 TCP 配置会影响连接的稳定性，具体在文章后面解释。

解析出的四个 108.157.229.* 裸 IP，反向 DNS（PTR）结果为 CloudFront 的 CDN 域名。这里本地和网站对四个 IP 的 PTR 结果都是 arn56.r.cloudfront.net，搜了下这是瑞典斯德哥尔摩 Stockholm, Sweden (ARN 是其 Arlanda 机场 IATA code)。因存在 CDN，不同地区的客户端去解析 files.vrchat.cloud 应该会得到不同的结果。

  • https://dnschecker.org/#PTR/108.157.229.115
  • https://mxtoolbox.com/SuperTool.aspx?action=ptr%3a108.157.229.115

有证据表明，该域名上的文件资源存储在 AWS S3 上：社区 API 文档提到了调用文件传输接口响应是 AWS S3 链接（猜测应该是pre-signed url）。

请求 https://s3.amazonaws.com/files.vrchat.cloud/ ，会看到这个 bucket 是存在的，从 HTTP header 里可见 x-amz-bucket-region: us-east-1，说明该 bucket 位于“声名显赫”的 us-east-1（位于美国弗吉尼亚），向其他 AWS 地区请求该 bucket（如 https://s3.eu-west-1.amazonaws.com/files.vrchat.cloud/ ），会收到“PermanentRedirect”响应，说明该 bucket 没开多地区备份策略如 Multi-Region Access Points。嗯…假如 us-east-1 down 了，VRC 玩家就加载不了世界和模型了吧。

#file-variants.vrchat.cloud

谷歌搜索 （和 AI 辅助搜索）搜不到该域名的用途，它只是不断出现在各帖子的代理规则配置中，同时该域名又确实是实际存在可解析的。看解析结果，该域名也套了 CloudFront，不过没有对应 S3 bucket https://s3.amazonaws.com/file-variants.vrchat.cloud。本地和网站解析结果理所当然得不同。

#玩家实时动作和语音

像其他在线游戏一样，玩家在 VRC 世界中的动作和语音，是通过 UDP 流量传输的，因为这类信息时效性较高，丢一部分也不需要重发。VRChat 借助名为 Photon Engine 的平台的资源来处理这部分数据传输。众所周知在 VRC 中创建世界实例时有四个地理位置选项，对应 Photon Engine 的四个地区集群。

根据这篇官方 FAQ4（和论坛贴），恼人的“VRChat 无响应（VRChat is not responding）”和“网络连接已超时（Your connection has timed out）”弹窗提醒电脑与 Photon Engine 服务器之间存在网络问题。

证据一：该文章和官方开端口文章都用了“real-time networking”字眼。证据二，日志铁证：

Error      -  [Behaviour] {{ NameServerAddress: 'ns.photonengine.io:27000', MasterServerAddress: 'gctok1018.exitgames.com:27001', GameServerAddress: '82.117.224.125:27002', Code: 'ClientTimeout', State: 'Disconnected', Time: '-1777227.993' }}
Debug      -  [Behaviour] Disconnected: ClientTimeout with Reason: 
Debug      -  [Behaviour] Connection summary: SCS v0 UDP SocketErrorCode: 0 WinSock
Error      -  [Behaviour] Timeout: 您与 VRChat 的连接已超时，请确认网络状态并重试。

搜索[5][6][7]得知，ns.photonengine.io，*.exitgames.com 两个域名在游戏客户端日志中出现，另外 Photon Engine 官方文档列出中国有个特殊的域名 ns.photonengine.cn。观察游戏日志和网络行为，只看到对 ns.photonengine.io 的请求：[Behaviour] Using nameserver host: ns.photonengine.io。

#通过代理玩 VRC

Connecting to VRChat requires a reliable connection to Cloudflare, AWS Cloudfront, and Photon Engine and a stable connection to download content such as Worlds and Avatars. – https://help.vrchat.com/hc/en-us/articles/360062658433-Troubleshooting-Connection-Issues-with-VRChat

为下载资源文件，游戏客户端将产生大量对端地址是 file-variants.vrchat.cloud 的入向流量，若该域名在你本地没有被污染，走直连可以节约代理流量。

若 Websocket API 域名（pipeline.vrchat.cloud）的出入流量走代理，则因代理客户端、代理服务器 TCP 超时配置和网络不稳定，可能出现 TCP 半开的情况，其中一种情况是：本地（代理客户端）到代理服务器间正常，但代理服务器到 pipeline 间网络不稳定，代理服务器不主动断开与客户端的连接，长时间等待 pipeline 服务器的响应。结果是本地迟迟等不到响应。

# https://clashparty.org/docs/guide/override/yaml
+rules:
  # --- vrchat start ---
  # 用端口判断 UDP 走直连，如果有问题（观察游戏日志），改成走代理
  # 其中 5055 5056 是官方文档中列出的
  # ref: https://help.vrchat.com/hc/en-us/articles/360062658793-Which-ports-do-I-need-to-open-for-VRChat
  # 5058 是社区列出的 Photon Engine 语音传输端口，27000-27100 是 Steam 要求打开的端口
  # ref: https://portforward.com/vrchat/
  # ref: https://help.steampowered.com/en/faqs/view/2EA8-4D75-DA21-31EB
  - 'AND,((NETWORK,UDP),(DST-PORT,5055)),DIRECT'
  - 'AND,((NETWORK,UDP),(DST-PORT,5056)),DIRECT'
  - 'AND,((NETWORK,UDP),(DST-PORT,5058)),DIRECT'
  - 'AND,((NETWORK,UDP),(DST-PORT,27000-27100)),DIRECT'
  # Photon Engine 域名直连（上面 UDP 端口规则应该会优先匹配，走不到这里）
  - 'DOMAIN-SUFFIX,exitgames.com,DIRECT'
  - 'DOMAIN-SUFFIX,photonengine.io,DIRECT'
  - 'DOMAIN-SUFFIX,photonengine.cn,DIRECT'
  # 其他域名走代理
  - 'DOMAIN,vrchat.com,Proxies'
  - 'DOMAIN,api.vrchat.cloud,Proxies'
  - 'DOMAIN,pipeline.vrchat.cloud,Proxies'
  - 'DOMAIN,files.vrchat.cloud,Proxies'
  - 'DOMAIN,file-variants.vrchat.cloud,Proxies'
  - 'DOMAIN,assets.vrchat.com,Proxies'
  # --- vrchat end ---
